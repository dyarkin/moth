# Getting Started

This guide creates one `git` module that manages `~/.gitconfig`.

## 1. Create a Module

```sh
moth module init git
```

This creates:

```text
~/.moth/git/
  manifest.yaml
  templates/
  variables/
  presets/
  snippets/
```

The module name is `git` because this module owns Git-related config.

## 2. Configure the Module Target Root

Create `~/.moth/config.yaml`:

```yaml
moduleRoots:
  git: ~
```

`moduleRoots.git` means files produced by the `git` module are placed under your home directory.

With this root, a template named `.gitconfig` becomes the target path `~/.gitconfig`.

## 3. Create Variables

Create `~/.moth/git/variables/main.yaml`:

```yaml
user:
  name: Example User
  email: user@example.com
```

Variables are YAML values available in templates and snippets.

## 4. Create a Snippet

Create `~/.moth/git/snippets/generated-header.txt`:

```text
# Generated by Moth. Edit the source in ~/.moth/git instead.
```

Snippets are reusable pieces of template text. They are not compiled into standalone target files.

## 5. Create a Template

Create `~/.moth/git/templates/.gitconfig`:

```ini
{% raw %}
{{ snippet("generated-header") }}

[user]
  name = {{ user.name }}
  email = {{ user.email }}
{% endraw %}
```

The template is almost the final config file. Moth replaces `{% raw %}{{ user.name }}{% endraw %}`, `{% raw %}{{ user.email }}{% endraw %}`, and the snippet call during compilation.

## 6. Compile the Module

```sh
moth compile git
```

This writes the rendered file to:

```text
~/.moth/git/.compiled/.gitconfig
```

At this point, no real config file has been changed yet.

## 7. Prepare the Root Compiled Tree

```sh
moth sync --prepare-only
```

This combines all module outputs into one tree under:

```text
~/.moth/.compiled
```

For this example, Moth prepares a file that maps to `~/.gitconfig`.

Use this step when you want to inspect generated files before applying symlinks.

## 8. Apply Symlinks

```sh
moth apply
```

`apply` creates symlinks from real target paths to files in `~/.moth/.compiled`.

You can also run the full workflow in one command:

```sh
moth sync
```

`moth sync` compiles modules, prepares the root `.compiled` tree, and applies symlinks.

## Optional: Add a Preset

Create `~/.moth/git/presets/work.yaml`:

```yaml
user:
  name: Example User
  email: user@company.com
```

Enable it:

```sh
moth module git preset enable work
```

Inspect the variables Moth will use:

```sh
moth module git vars
```

Then run `moth sync` to regenerate and apply the config with the preset value.

## Safety

Moth does not overwrite existing files, directories, or unexpected symlinks. If `~/.gitconfig` already exists and is not managed by Moth, `apply` reports a conflict instead of replacing it.
